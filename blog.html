Answering Some Question:

Discuss the scope of var, let, and const.

var : In JavaScript "var" is a function-scoped. Because, if we declare the "var" in a function we can only use it inside the function. On the other hand if we declare "var" outside of a function we can globally use it any where of the program.

let : In JavaScript "let" is a block-scoped. Because, if we declare the "let" in a function we can only use it inside the function.We can not use "let" outside of a function or globally any where of the program.

const : In JavaScript "const" is a block-scoped. Because, if we declare the "const" in a function we can only use it inside the function.We can not use "const" outside of a function or globally any where of the program.

Tell us the use cases of null and undefined.

null : In JavaScript "null" primitive value. We often use this to indicate that a value of an object, property, array or any variable that has no value. That time we explicitly set the "null".

undefined : In JavaScript "undefined" also a primitive value. Sometimes we declared a variable but did not assign any value. Or we declare a function but there is no return statement that time it automatically assigned "undefined".

What do you mean by REST API.

REST API: In JavaScript "REST" full form is Representational State Transfer and "API" stands for Application Programming Interface. API is a set of rules that defines how the devices can connect and communicate with the web server. And the REST api is a specific type of api that conform to the constraints of the REST architecture. Overall we can say that "REST API" is a way to connect and communicate with web server easily.
